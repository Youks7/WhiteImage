# 不会PS修图，也能批量做干净的墨镜白底图

> 给原图文件夹 → 确认一张样图 → AI 逐张精修、检查并输出 PNG
>
> 作者：[秋水Kero](https://x.com/Isonlyonenice)

---

## 这是什么

拍出来的墨镜镜片有灰尘、灯箱反光、房间倒影，白底下面还有一大片商品倒影？

**把原图、输出目录和质量参考图交给 Codex，WhiteImage 会按固定流程完成批量白底精修。**

它不是给所有商品套同一种颜色的滤镜，也不是普通抠图换白底。每张输入原图始终是该商品结构、镜片颜色、透明度、拍摄角度和构图的唯一依据。

整个流程分三步：

### ① 给原图文件夹 → 先统计，不直接开跑

Skill 会递归读取所有子文件夹，统计图片总数、检查同名输出冲突，并整理镜片类型与拍摄视角。

原图不会被覆盖。最终输出保持原来的子目录和基本文件名，只把图片统一保存为 PNG。

### ② 做一张代表样图 → 你确认效果标准

先选择一张有代表性的原图，用内置图像编辑能力生成样图。样图只用来确认：

- 镜片是否零污点、零环境或灯具倒影；
- 镜片渲染是否干净、连续、有层次；
- 背景是否为均匀纯白；
- 商品下方是否没有可识别的倒影或灰色光带；
- 商品细节是否清楚、自然，没有明显 AI 重绘感。

只有你明确确认后，才会继续处理剩余图片。

### ③ 批量逐张精修 → 逐张检查，不只核对数量

每张图片都以自己的原图为编辑目标，单独调用图像编辑。完成后依次检查结构、颜色、透明度、镜片洁净度、白底、主体下方处理和真实细节。

文件可读取、数量一致，只代表批次完整；**不代表图片视觉合格**。发现色偏、残留反光、结构漂移或主体倒影时，必须从原图重新处理。

## 它主要解决什么

- 清除镜片上的灰尘、污点、划痕、雾气、白点、反射线和环境倒影；
- 去掉软箱、摄影器材、摄影师、房间和其他可识别物体的反射；
- 保留黄色透明、黑灰透明、橙金偏光、蓝色、绿色和银色镀膜各自的原色；
- 避免把“无环境反光”错误处理成没有层次的纯色镜片；
- 生成干净、均匀的棚拍纯白背景；
- 去掉主体下方的商品倒影、灰色反光带、地面高光和大片光晕；
- 锁定镜框、镜片、鼻梁、镜腿、铰链、通风孔和装饰件；
- 保留原有角度、构图、比例、位置、方向、材质和颜色；
- 删除文字、型号、标签、贴纸和水印，不添加新标志；
- 镜像原目录结构输出 PNG，并核对数量、缺失文件、额外文件和实际尺寸。

---

## 镜片颜色为什么不会统一套样图

样图和参考图只能定义“多干净、多清楚、白底怎么处理”，不能替其他商品决定颜色。

为降低批量色偏，Skill 会遵守四条规则：

1. 每次编辑前重新观察当前原图，不按文件夹名称猜颜色。
2. 分别记录当前镜片的色相、明暗、饱和度、透明度和视角镀膜变化。
3. 透明黄、透明黑灰、银色偏光和彩色镀膜先分别校准，再进入批量处理。
4. 返工永远从原图重新开始，不把失败成品继续当底图，避免颜色和结构逐轮漂移。

这套机制能显著降低颜色被样图带偏的概率，但 AI 图片编辑仍可能出现单张偏差，所以样图确认和逐张验收不能省略。

## 适合谁用

- 墨镜、骑行眼镜、运动眼镜和普通眼镜商家；
- 需要批量制作电商白底图的运营或摄影团队；
- 手上有多颜色、多镀膜、多角度原图，需要统一质量标准的人；
- 不会写复杂生图提示词，但能判断商品颜色和结构是否准确的人。

> 当前 Skill 专门针对墨镜和眼镜类商品，不是通用的全品类白底图工具。

## 怎么使用

### 第一步：开始任务并生成样图

安装后，在 Codex 新任务中先粘贴：

```text
使用 $refine-sunglasses-white-background 批量精修墨镜白底图。

输入目录：[原图目录]
输出目录：[输出目录]
合格效果参考目录：[参考图目录]

先递归统计所有图片，再选择一张有代表性的样图。
样图得到我确认后，才处理剩余全部图片。

每张原图是对应商品款式、镜片颜色、透明度、角度和构图的唯一依据。
参考图只决定镜片洁净度、商业渲染质量、白底、清晰度和主体下方处理标准。
完成后逐张检查，并核对输入数量与输出数量一致。
```

### 第二步：确认样图后完成整批

样图合格后，在同一个 Codex 任务中继续发送：

```text
这张样图合格。请把它固定为本批次的效果标准，但只沿用镜片洁净度、零环境反光、干净受控的商业渲染渐变、纯白背景、商品清晰度和主体下方无倒影的处理方式。

现在继续处理输入目录中的全部剩余图片，直到整批完成，不要每张等待我确认。保留已确认样图，不要重复处理。

每张原图仍是自身款式、镜片原色、透明度、镀膜表现、材质、角度和构图的唯一依据。绝对不要把样图的镜片颜色、透明度、款式或角度套给其他图片。

每张完成后都要对照其原图逐张验收；不合格图片必须从对应原图重新处理。全部完成后运行批次核验，确认输入图片总数与合格 PNG 输出总数一致，并报告缺失文件、额外文件、不可读取文件、实际尺寸分布和返工数量。
```

然后按顺序完成：

1. 提供原图目录、输出目录和参考图目录；
2. 检查 Codex 报告的图片总数和输出风险；
3. 查看并确认第一张样图；
4. 等待剩余图片逐张处理和验收；
5. 查看最终数量、尺寸、返工记录和视觉检查报告。

> 最适合在能访问本机文件并带有内置图像生成/图像编辑能力的 Codex 环境中使用。批次检查脚本使用 PowerShell，Windows 环境最直接。

## 它怎么兼顾速度和质量

样图及各镜片类型完成颜色校准后，Skill 最多同时处理两张互不相关的图片，以减少等待时间。每张图片仍使用独立原图、独立编辑请求和独立验收。

以下情况会自动收紧为串行处理：

- 透明黄、银色、黑灰透明等颜色较容易漂移；
- 同一类型连续两次出现色偏；
- 输入与输出的映射不够明确；
- 小批次检查发现结构或镜片质感不稳定。

并发只能减少等待，不能减少图片编辑次数和检查步骤。实际耗时会受到图片数量、服务排队和返工次数影响，项目不承诺固定完成时间。

## 图片不满意时怎么返工

不需要把整批从头再做。先指出具体图片和具体问题：

- 镜片颜色不对 → 对照该张原图修正色相、明暗、饱和度和透明度；
- 镜片仍有反射线或环境倒影 → 只收紧镜片清洁与受控渐变要求；
- 镜框、镜腿或开孔变形 → 强制回到原图，重新锁定结构；
- 主体下方仍有倒影或灰带 → 重新处理白底与接触阴影；
- 数量、命名或目录不一致 → 重新运行批次核验脚本定位缺失或冲突。

返工时不要拿失败成品继续生成。始终以对应原图作为商品事实来源。
