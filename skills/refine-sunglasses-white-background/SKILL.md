# 墨镜白底批量精修

使用内置图像生成/图像编辑能力逐张编辑商品图。禁止用脚本、抠图、阈值、填色或重绘代替模型编辑；本地脚本只能建立清单、建立目录、复制/命名和检查结果。

开始前完整阅读：

- `references/quality-standard.md`：不可放宽的质量与文件标准。
- `references/prompt-templates.md`：逐张编辑、颜色返工和验收模板。

## 约束优先级

发生冲突时按以下顺序执行：

1. 每张原图是商品结构、可见颜色、镜片透明度、拍摄角度和构图的唯一事实依据。
2. 用户确认的样图只定义镜片洁净度、可控渲染质感、白底、清晰度和主体下方处理方式。
3. 其他参考图只定义质量下限，绝不能提供款式、颜色、角度或比例。
4. 文件格式、目录结构和命名规则。

不得因“去反光”把透明镜片变成不透明色块，也不得把银色、黄色、黑灰、蓝色或绿色镜片套用样图的饱和度。不得根据文件夹名称替代肉眼读取原图；同一镀膜的正面、背面和斜视颜色可能不同。

## 执行流程

### 1. 预检与清单

1. 确认输入、输出和参考目录；不覆盖输入原图。
2. 运行 `scripts/check_batch.ps1 -Mode Inventory`，递归统计图片并检查输出名冲突。不得用手工浏览或临时脚本替代该脚本结果；报告时引用其返回的 `InputImageCount`、`OutputCollisions`、`ExistingExpectedOutputCount`、`ExistingOtherFileCount` 和 `SafeToWrite`。若其他检查与这些字段矛盾，停止并重跑官方脚本。
3. 按镜片颜色/光学类型和视角整理清单，但只将分组用于调度与抽查。
4. 若 `ExistingExpectedOutputCount` 大于 0 或 `SafeToWrite` 为 false，明确说明覆盖/目录风险。若用户要求清空输出目录，只在明确授权后执行；先解析并复核精确绝对路径，不触碰输入目录或其他目录。

### 2. 样图确认

1. 选择具有代表性的原图；如用户已指定则服从用户选择。
2. 将原图标为“编辑目标”，将参考图标为“质量参考”。
3. 使用内置编辑能力生成一张样图并与原图、参考图并排检查。
4. 等待用户明确确认后再处理剩余图片。
5. 将已确认样图固定为效果标准，但绝不把它的镜片颜色或不透明度迁移到其他商品。

如果批次同时包含透明黄、透明黑灰、银色偏光和彩色镀膜等明显不同类型，在全面批量前先对每种光学类型做内部颜色校准。发现任何色偏时暂停该类型，不扩散错误。

### 3. 逐张编辑

1. 每张图片单独调用一次内置图像编辑；不得把多件商品合成在同一次请求中。
2. 每次调用前加载并观察该原图。输入角色必须写清：原图是结构/颜色/透明度权威；样图与参考图仅是洁净度和白底标准。
3. 使用 `references/prompt-templates.md` 中的主模板，并填入该图实际可见的镜片色相、亮度、饱和度、透明度及视角镀膜变化。
4. 始终从原图重新编辑。返工时不要以失败成品作为新的结构或颜色来源，避免逐轮漂移。
5. 输出为 PNG，保持相对目录和基本文件名，仅更换扩展名。

### 4. 保守提速

样图确认后，可在运行环境支持时最多同时处理两张互不相关的图片。必须满足：

- 每张仍使用独立原图和独立提示；
- 维护明确的输入—生成结果—目标路径映射；
- 每完成一小批立即验收，不积压整批后才发现系统性色偏；
- 透明黄、银色、黑灰透明等曾出现色偏或难以判断的类型先串行校准；
- 出现同类色偏两次，立即停止该类型并收紧提示，不能继续并发。

并发只是减少等待时间，不能减少图像调用次数或检查步骤。不要承诺固定耗时，服务端排队和返工率不可控。

### 5. 逐张验收

逐张将输出与其原图并排检查，顺序固定为：

1. 结构、角度、构图和部件；
2. 镜片色相、明暗、饱和度、透明度及正反面镀膜差异；
3. 镜片污点、反射线、环境/灯具倒影和高光斑；
4. 纯白背景、主体下方倒影和光晕；
5. 边缘、材质、开孔、铰链和全景深清晰度。

任一项不合格即使用针对性模板重新从原图生成。颜色不合格时，先修颜色和透明度，不要同时增加新的创意性描述。

禁止仅凭文件数量、PNG 格式或可读取性宣布视觉合格。

### 6. 最终核验与报告

1. 运行 `scripts/check_batch.ps1 -Mode Validate`。
2. 确认输入数等于预期输出数、缺失为 0、额外文件为 0、不可读取 PNG 为 0。
3. 抽查并覆盖每种镜片类型及正面、背面、侧面、45度视角；如果用户要求逐张检查，则保留逐张检查结论。
4. 报告输入总数、输出总数、输出目录、PNG 格式、实际原生尺寸分布、返工数量和视觉验收结果。
5. 若仍有失败项，列出相对路径和失败原因；不得把“已生成”写成“已合格”。
